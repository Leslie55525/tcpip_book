1、请说明 TCP 套接字连接设置的三次握手过程。尤其是 3 次数据交换过程每次收发的数据内容。
    第一次A发送初始的SEQ，请求关键字是SYN，表示收发数据前同步传输的消息，此时报文中的ACK为空。B收到报文
后向A发送一个新的SEQ表示自己的序号，然后ACK代表已经接受到了A的数据，希望接受下一个消息。A收到B的确认
后，还需要给B发送一次SEQ和ACK表示自己收到了B的消息。

2、TCP 是可靠的数据传输协议，但在通过网络通信的过程中可能丢失数据。请通过 ACK 和 SEQ 说明 TCP 通过
和何种机制保证丢失数据的可靠传输。
    通过超时重传的机制来保证。如果报文发出的时间在特定的时间内，发送消息的主机没有收到另一个主机的回复，
那么就会继续发送这条消息，直到收到回复为止。

3、TCP 套接字中调用 write 和 read 函数时数据如何移动？结合 I/O 缓冲进行说明。
    write写入到输出缓冲，在适当的时候传送到对方的输入缓冲中。对方使用read会从自己的输入缓冲中读取。

4、对方主机的输入缓冲剩余 50 字节空间时，若本主机通过 write 函数请求传输 70 字节，请问 TCP 如何处理
这种情况？
    TCP中有滑动窗口控制协议，所以传输的时候会保证传输的字节数小如等于自己能接受的字节数。
